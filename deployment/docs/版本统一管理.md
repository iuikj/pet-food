# 版本统一管理方案

> **核心原则：单一真实来源（Single Source of Truth）= Git Tag**

---

## 设计目标

打一个 git tag → 所有组件版本号自动同步：

| 组件 | 文件 | 写入方式 |
|------|------|----------|
| 后端 Python 包 | `pyproject.toml` | CI sed 替换 |
| 后端运行时 | `src/__version__.py` | CI sed 替换 |
| 后端 API | `/health` 返回值 | 读取 `__version__` |
| 后端 OpenAPI | FastAPI `version=` | 读取 `__version__` |
| 前端 npm 包 | `package.json` | CI 用 node 修改 |
| 前端运行时 | Vite `__APP_VERSION__` | 编译期读取 `package.json` |
| Android | `versionName` / `versionCode` | Gradle `-P` 参数注入 |

---

## 一键发布流程

```bash
# 1. 在后端仓库打 tag（触发 CD 部署 + APK 构建）
cd pet_food_backend/pet-food
git tag -a v1.2.3 -m "Release v1.2.3"
git push origin v1.2.3

# 2. 在前端仓库打相同 tag（CD 需要 checkout 这个 tag）
cd ../../frontend/web-app
git tag -a v1.2.3 -m "Release v1.2.3"
git push origin v1.2.3
```

CI 自动：
1. 后端镜像构建前 → `sync-version.sh 1.2.3 backend`
2. 前端镜像构建前 → `sync-version.sh 1.2.3 frontend`
3. APK 构建前 → 计算 versionCode = `1*10000 + 2*100 + 3 = 10203`
4. 部署到生产，前端可调用 `/health` 验证后端版本

## 版本号规则

### 语义化版本（SemVer）

```
MAJOR.MINOR.PATCH (例如 1.2.3)
  │     │     └─ 向下兼容的 bug 修复
  │     └─────── 向下兼容的功能新增
  └───────────── 不向下兼容的重大变更
```

### Android versionCode 计算

```python
versionCode = MAJOR * 10000 + MINOR * 100 + PATCH

# 示例
v1.0.0   → 10000
v1.2.3   → 10203
v2.0.0   → 20000
v9.99.99 → 99999  # 上限，足够 100 年发布
```

**约束：** Android Play Store 要求每次发布 versionCode 必须递增，本规则保证语义化版本永远递增。

## 验证版本一致性

发布后访问以下端点验证：

```bash
# 后端版本
curl http://你的服务器/health
# 期望：{"data": {"version": "1.2.3"}}

# 前端版本（在浏览器控制台）
console.log(__APP_VERSION__)
# 期望：1.2.3

# Android 版本（手机设置 → 应用信息）
# 期望：版本 1.2.3 (10203)
```

## 本地开发版本号

本地开发时**不需要**手动维护版本号，保持文件中的初始值即可：

- `pyproject.toml`: `version = "0.1.0"`
- `package.json`: `"version": "0.0.0"`
- `__version__.py`: `__version__ = "0.1.0"`

只有 CI 在打 tag 时才会改这些文件（且不提交回仓库，仅用于构建产物）。

## 故障排查

### 后端版本和前端不一致

**症状：** `/health` 返回 1.2.3，但前端显示 0.0.0

**原因：** 前端 tag 没推送，CD 仍用旧 tag 构建

**解决：**
```bash
cd frontend/web-app
git tag -a v1.2.3 -m "Release v1.2.3"
git push origin v1.2.3
# 然后重新触发 CD：手动 dispatch 或重推后端 tag
```

### Android 版本号没更新

**症状：** APK 安装后版本仍显示 1.0

**原因：** workflow 没传 `-P` 参数，`build.gradle` 用了默认值

**检查：** workflow 的 `Build Android APK` 步骤有：
```yaml
./gradlew assembleRelease \
  -PappVersionName="1.2.3" \
  -PappVersionCode="10203"
```

### Play Store 拒绝上传：versionCode 必须 > 上次

**原因：** 重复发同一版本号

**解决：** 不要回退 tag，直接发新 patch（如 v1.2.4）

## 关键文件清单

```
pet_food_backend/pet-food/
├── scripts/
│   └── sync-version.sh             # 核心同步脚本
├── src/
│   ├── __version__.py              # 后端版本单一来源
│   └── api/main.py                 # 引用 __version__ 用于 FastAPI 和 /health
├── pyproject.toml                  # CI 自动同步
└── .github/workflows/
    ├── cd-production.yml           # 调用 sync-version.sh backend/frontend
    └── build-android.yml           # 调用 sync-version.sh android

frontend/web-app/
├── vite.config.js                  # define __APP_VERSION__
├── src/utils/version.ts            # 暴露 APP_VERSION 给业务代码
├── package.json                    # CI 自动同步
└── android/app/build.gradle        # 通过 -PappVersionName 接受 CI 注入
```
