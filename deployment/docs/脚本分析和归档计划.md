## ✅ 保留的脚本（8 个 → 6 个）

### 1. deploy-ghcr.sh ✅
**用途：** GitHub Actions CD 部署脚本  
**使用场景：** 
- GitHub Actions 自动调用
- 服务器上手动部署

**路径依赖：** 无（使用相对路径）

---

### 2. rollback-ghcr.sh ✅
**用途：** CD 回滚脚本  
**使用场景：** 
- 部署失败自动回滚
- 手动回滚到指定版本

**路径依赖：** 无

---

### 3. deploy.sh ✅
**用途：** 本地/生产一键部署脚本  
**使用场景：** 
- 本机开发测试
- 手动部署到生产环境

**路径依赖：** 
```bash
COMPOSE_FILE="$SCRIPT_DIR/docker-compose.prod.yml"
ENV_FILE="$SCRIPT_DIR/.env.prod"
```
✅ 无影响（使用相对路径）

---

### 4. deploy.ps1 ✅
**用途：** Windows PowerShell 部署脚本  
**使用场景：** 
- Windows 本机开发测试

**路径依赖：** 
```powershell
$ComposeFile = Join-Path $ScriptDir "docker-compose.prod.yml"
$EnvFile     = Join-Path $ScriptDir ".env.prod"
```
✅ 无影响（使用相对路径）

---

### 5. entrypoint.sh ✅
**用途：** API 容器入口脚本  
**使用场景：** 
- Docker 容器启动时自动执行
- 等待数据库 → 运行迁移 → 启动 API

**路径依赖：** 无（容器内路径）

---

### 6. server-setup.sh ✅
**用途：** 服务器环境准备脚本  
**使用场景：** 
- 新服务器初始化
- 配置 Docker、镜像加速、目录结构

**路径依赖：** 
```bash

DEPLOYMENT_DIR="$PROJECT_DIR/deployment"
```
✅ 无影响（绝对路径）

### 文档路径变化
- ❌ **无影响** - 所有脚本都不引用文档路径

### Nginx 配置路径
```yaml

  - ./deployment/nginx/nginx.conf:/etc/nginx/nginx.conf:ro
```

- ❌ **无影响** - Nginx 配置路径未变化

### 脚本相对路径
所有活跃脚本都使用相对路径或绝对路径：
- `deploy.sh` - 使用 `$SCRIPT_DIR` 相对路径
- `deploy.ps1` - 使用 `$ScriptDir` 相对路径
- `deploy-ghcr.sh` - 使用相对路径

✅ **结论：文档重组不影响任何脚本功能**
