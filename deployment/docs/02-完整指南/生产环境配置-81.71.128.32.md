## 📝 待完成的配置步骤

### 步骤 1：生成 SSH 部署密钥

**在你的本地 Windows 机器上执行：**

```bash
# 打开 Git Bash 或 PowerShell

# 生成专用的部署密钥
ssh-keygen -t ed25519 -C "github-actions-deploy@petfood" -f ~/.ssh/petfood_deploy_key

# 查看私钥（稍后复制到 GitHub Secrets）
cat ~/.ssh/petfood_deploy_key

# 查看公钥（稍后添加到服务器）
cat ~/.ssh/petfood_deploy_key.pub
```

**在服务器上执行：**

```bash
# SSH 登录服务器
ssh root@81.71.128.32

# 添加公钥到 authorized_keys
echo "ssh-ed25519 AAAA... github-actions-deploy@petfood" >> ~/.ssh/authorized_keys

# 设置权限
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh

# 测试连接（在本地执行）
ssh -i ~/.ssh/petfood_deploy_key root@81.71.128.32 "echo 'SSH 连接成功'"
```

### 步骤 2：配置 GitHub Secrets

访问：`https://github.com/iuikj/pet-food/settings/secrets/actions`

点击 **New repository secret**，逐个添加以下 Secrets：

#### 必需的 Secrets（7 个）

| Secret 名称 | 值 | 说明 |
|------------|---|------|
| `SERVER_HOST` | `81.71.128.32` | 服务器 IP |
| `SERVER_USER` | `root` | SSH 用户名 |
| `SERVER_PORT` | `22` | SSH 端口（默认） |
| `SERVER_SSH_KEY` | `[步骤 1 生成的私钥内容]` | 完整的私钥，包括 `-----BEGIN` 和 `-----END` |
| `SERVER_APP_DIR` | `/opt/petfood/pet_food_backend/pet-food` | 服务器部署目录 |
| `GHCR_USERNAME` | `iuikj` | 你的 GitHub 用户名 |
| `GHCR_TOKEN` | `&lt;YOUR_GITHUB_PAT&gt;` | 你刚才测试用的 GitHub PAT |

#### 前端仓库访问 Token（1 个）

| Secret 名称 | 值 | 说明 |
|------------|---|------|
| `FRONTEND_REPOSITORY_TOKEN` | `[GitHub PAT]` | 用于 checkout 前端私有仓库 |

**如何生成 `FRONTEND_REPOSITORY_TOKEN`：**

1. 访问：`https://github.com/settings/tokens`
2. 点击 **Generate new token (classic)**
3. 勾选权限：
   - ✅ `repo` - 访问私有仓库
4. 生成后复制 Token

**注意：** 如果前端仓库是公开的，可以跳过这个 Secret。

### 步骤 3：配置 GitHub Variables（可选）

访问：`https://github.com/iuikj/pet-food/settings/variables/actions`

点击 **New repository variable**，添加前端构建参数：

| Variable 名称 | 值 | 说明 |
|--------------|---|------|
| `VITE_API_BASE_URL` | `http://81.71.128.32/api/v1` | 前端 API 基址 |
| `VITE_ENABLE_SSE` | `true` | 启用 SSE 流式 |
| `VITE_RECONNECT_DELAY` | `3000` | SSE 重连延迟（毫秒） |

**如果不配置，会使用默认值。**

### 步骤 4：服务器准备

**在服务器上执行：**

```bash
# 1. 创建状态目录

# 5. 验证登录成功
docker pull ghcr.io/iuikj/hello-world:latest
```

## 🔍 故障排查

### 问题 1：GitHub Actions 构建失败

**检查步骤：**

1. 查看 Actions 日志，找到具体错误
2. 常见问题：
   - ❌ Dockerfile 路径错误
   - ❌ 依赖安装失败
   - ❌ 构建超时

**解决方案：**
- 检查 Dockerfile 是否正确
- 检查网络连接
- 增加构建超时时间

### 问题 2：前端仓库 checkout 失败

**错误信息：**
```
Error: Resource not accessible by integration
```

**解决方案：**
- 确保 `FRONTEND_REPOSITORY_TOKEN` 已配置
- 确保 Token 有 `repo` 权限
- 确保前端仓库存在且有对应的 tag

### 问题 3：SSH 连接失败

**错误信息：**
```
ssh: connect to host 81.71.128.32 port 22: Connection refused
```

**解决方案：**
- 检查 `SERVER_SSH_KEY` 是否正确（包含完整的私钥）
- 检查服务器 SSH 服务是否运行
- 检查防火墙规则

### 问题 4：部署脚本执行失败

**检查服务器日志：**

```bash
# 查看详细日志
tail -100 /opt/petfood/pet_food_backend/pet-food/deployment/.cd-state/deploy-ghcr.log

# 查看容器日志
docker logs pet-food-api --tail 100
docker logs pet-food-frontend --tail 100
```

**常见问题：**
- ❌ 镜像拉取失败 → 检查 GHCR 登录
- ❌ 健康检查超时 → 检查 API 启动日志
- ❌ 环境变量缺失 → 检查 `.env.prod`
