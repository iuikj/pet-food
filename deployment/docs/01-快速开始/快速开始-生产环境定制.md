## 📋 5 步完成配置

### 步骤 1：生成 SSH 密钥（2 分钟）

**在本地 Windows 机器（Git Bash）：**

```bash
# 生成密钥
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/petfood_deploy_key

# 查看私钥（稍后用）
cat ~/.ssh/petfood_deploy_key

# 查看公钥
cat ~/.ssh/petfood_deploy_key.pub
```

**在服务器上：**

```bash
# 添加公钥
echo "ssh-ed25519 AAAA... github-actions-deploy" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

---

### 步骤 2：服务器准备（3 分钟）

**方式 A：使用自动化脚本（推荐）**

```bash
# 在本地上传脚本
scp pet_food_backend/pet-food/deployment/server-setup.sh root@81.71.128.32:/tmp/

# SSH 登录服务器
ssh root@81.71.128.32

# 执行脚本
bash /tmp/server-setup.sh

# 按提示输入：
# - GitHub 用户名: iuikj
# - GitHub Token: &lt;YOUR_GITHUB_PAT&gt;
```

**方式 B：手动配置**

```bash
# SSH 登录服务器
ssh root@81.71.128.32

# 创建目录
mkdir -p /opt/petfood/pet_food_backend/pet-food/deployment/.cd-state

# 停止旧 webhook
sudo systemctl stop petfood-webhook || true
sudo systemctl disable petfood-webhook || true

# 登录 GHCR
echo "&lt;YOUR_GITHUB_PAT&gt;" | docker login ghcr.io -u iuikj --password-stdin
```

### 步骤 3：配置 GitHub Secrets（5 分钟）

访问：`https://github.com/iuikj/pet-food/settings/secrets/actions`

点击 **New repository secret**，添加以下 7 个 Secrets：

| Secret 名称 | 值 |
|------------|---|
| `SERVER_HOST` | `81.71.128.32` |
| `SERVER_USER` | `root` |
| `SERVER_PORT` | `22` |
| `SERVER_SSH_KEY` | `[步骤 1 的私钥完整内容]` |
| `SERVER_APP_DIR` | `/opt/petfood/pet_food_backend/pet-food` |
| `GHCR_USERNAME` | `iuikj` |
| `GHCR_TOKEN` | `&lt;YOUR_GITHUB_PAT&gt;` |

**前端仓库访问（如果是私有仓库）：**

| Secret 名称 | 值 | 如何生成 |
|------------|---|----------|
| `FRONTEND_REPOSITORY_TOKEN` | `[GitHub PAT]` | https://github.com/settings/tokens → Generate new token → 勾选 `repo` |

## 🔍 常见问题

### Q1: GitHub Actions 构建失败

**检查：**
- 查看 Actions 日志中的具体错误
- 确认 Dockerfile 路径正确
- 确认依赖安装成功

### Q2: 前端仓库 checkout 失败

**错误：** `Resource not accessible by integration`

**解决：**
- 确保 `FRONTEND_REPOSITORY_TOKEN` 已配置
- 确保 Token 有 `repo` 权限
- 确保前端仓库有对应的 tag

### Q3: SSH 连接失败

**错误：** `Connection refused`

**解决：**
- 检查 `SERVER_SSH_KEY` 格式（包含完整的 `-----BEGIN` 和 `-----END`）
- 检查服务器 SSH 服务状态
- 测试本地 SSH 连接：`ssh -i ~/.ssh/petfood_deploy_key root@81.71.128.32`

### Q4: 部署脚本执行失败

**检查服务器日志：**
```bash
tail -100 /opt/petfood/pet_food_backend/pet-food/deployment/.cd-state/deploy-ghcr.log
```

**常见原因：**
- GHCR 登录失败 → 检查 Token
- 健康检查超时 → 检查 API 日志
- 环境变量缺失 → 检查 `.env.prod`
