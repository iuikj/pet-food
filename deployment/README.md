## 🎯 快速开始

### 新手入门（推荐）

```bash
# 1. 阅读快速开始指南
cat deployment/docs/01-快速开始/快速开始指南.md

# 2. 按照 5 步完成配置

git push origin v1.0.0
```

```bash

git tag -a v1.1.0 -m "Release v1.1.0: 新功能"
git push origin v1.1.0

# 查看部署进度
# 访问：https://github.com/你的用户名/pet-food/actions

# 验证部署
curl http://你的服务器IP/health
```

## 🔑 核心特性

### GitHub Actions CD 方案

- ✅ **自动化部署** - 推送标签自动触发部署
- ✅ **镜像加速** - 使用国内镜像源，拉取速度快
- ✅ **零停机部署** - Docker 滚动更新
- ✅ **自动回滚** - 部署失败自动回滚
- ✅ **健康检查** - 确保服务可用才完成部署

### 为什么选择 GitHub Actions？

相比 Webhook 方案：

| 特性 | GitHub Actions | Webhook |
|------|----------------|---------|
| **安全性** | ✅ SSH 单向连接 | ❌ 需要暴露端口 |
| **可靠性** | ✅ GitHub 基础设施 | ❌ 依赖服务器稳定性 |
| **功能** | ✅ 并行构建、缓存 | ❌ 功能受限 |
| **维护** | ✅ 配置在代码仓库 | ❌ 需要维护服务 |

## ⚠️ 重要提示

### 前后端标签同步

**问题：** 前端仓库没有对应标签会导致部署失败

**解决方案：** 前后端打相同的标签

```bash
# 后端仓库
cd pet_food_backend/pet-food
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0

# 前端仓库
cd frontend/web-app
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

### 环境变量配置

**必须配置的环境变量：**

- `JWT_SECRET_KEY` - JWT 密钥
- `POSTGRES_PASSWORD` - 数据库密码
- `REDIS_PASSWORD` - Redis 密码
- `MINIO_ROOT_PASSWORD` - MinIO 密码
- `DASHSCOPE_API_KEY` - LLM API 密钥
- `TAVILIY_API_KEY` - 搜索 API 密钥

**生成随机密钥：**

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

## 🆘 获取帮助

### 遇到问题？

1. **查看故障排查** - [故障排查手册](./docs/04-故障排查/故障排查手册.md)
2. **查看完整指南** - [完整部署指南](./docs/02-完整指南/完整部署指南.md) 的故障排查章节
3. **查看 GitHub Actions 日志** - 访问 Actions 页面查看详细错误
4. **查看服务器日志** - SSH 到服务器查看容器日志

### 常见问题

- **部署失败** → 查看 GitHub Actions 日志和服务器日志
- **健康检查超时** → 检查 API 容器日志和数据库连接
- **镜像拉取失败** → 配置镜像加速或检查网络
- **SSH 连接失败** → 验证 SSH 密钥配置
